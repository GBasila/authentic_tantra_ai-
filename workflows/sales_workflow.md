# Sales Workflow

## Inbound inquiry (DM / email)

1. **T+0:** Community Manager or Ops Manager receives inquiry
2. **T+1h:** Inquiry tagged by segment + stage (Stranger → ... → Devotee)
3. **T+2h:** Sales Agent generates response suggestion (3 variants)
4. **T+4h:** Human reviews + sends personalized response
5. **T+24h:** If no reply, soft follow-up (one, not a sequence)
6. **T+72h:** If no reply, close thread. Do not press.

## High-ticket application (Retreat / 1:1)

1. **T+0:** Application submitted
2. **T+24h:** Reviewed by Retreat Director or Ops Manager
3. **T+48h:** Discovery call scheduled if fit
4. **T+7d:** Discovery call (30 min) — human, not Victoria
5. **T+8d:** Decision: accept / waitlist / gentle release
6. **T+9d:** Communication sent — warm regardless of outcome

## Launch workflow
See `/sales/launch_sequence.md` for 8-week cycle.

## Daily during launch week (cart open)
- 06:00 · Email sent
- 09:00 · IG post published
- 11:00 · Community Manager answers last 24h DMs with Sales Agent suggestions
- 14:00 · Victoria 15-min voice check-in (is the container holding?)
- 19:00 · Stories posted from Victoria

## Guardrails
- Sales Agent never auto-sends
- Victoria does not respond to DMs directly during launch (boundaries)
- No sales conversation happens on retreat
- Scholarship applications reviewed weekly, not in-thread

## Post-launch
1. Cart closes
2. Team decompresses 48h
3. Retrospective meeting
4. Write `/sales/launches/YYYY-QN-offer/retrospective.md`
5. Feed insights back to Audience Agent + CEO Agent
